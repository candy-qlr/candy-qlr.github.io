  • 一个基于 Vue.js 生态的第三方开源服务端渲染应用框架

  • 它可以帮我们轻松的使用 Vue.js 技术栈构建同构应用

  • Github 仓库open in new window

  • nuxtjs 预设了利用 Vue.js 开发服务端渲染的应用所需要的各种配置

  • 动态路由

    pages/_slug.vue

    pages/_continent/_mountain.vue

    当您不知道页面名称（因为它来自 API）或者您不想一遍又一遍地创建相同的页面时，可以创建动态页面。要创建动态页面，您需要在 .vue 文件名或目录名之前添加下划线，如果您希望目录是动态的。您可以随意命名文件或目录，但必须用下划线作为前缀。

    如果您定义了_slug.vue在 pages 文件夹中命名的文件，则可以使用带有 params.slug 的上下文访问该值

    // pages/_slug.vue
<template>
<template>
  <h1>{{ this.slug }}</h1>
</template>

<script>
  export default {
    async asyncData({ params }) {
      const slug = params.slug // When calling /abc the slug will be "abc"
      return { slug }
    }
  }
</script>

# 3. 目录结构

  • pages

    该 pages 目录包含您的应用程序视图和路由。Nuxt.js 读取.vue 此目录中的所有 文件并自动为您创建路由器配置。

    每个 Page 组件都是一个 Vue 组件，但 Nuxt.js 添加了特殊的属性和功能，使您的通用应用程序的开发尽可能简单。

  • components

    该 components 目录是您放置所有 Vue.js 组件的地方，然后将这些组件导入到您的页面中。

    使用 Nuxt.js，您可以创建组件并将它们自动导入到 .vue 文件中，这意味着无需在脚本部分手动导入它们。一旦您将组件设置为 true，Nuxt.js 将扫描并自动导入这些组件。

  • assets

    该 assets 目录包含未编译的资产，例如样式、图像或字体。

    图片：

    在您的 vue 模板中，如果您需要链接到您的 assets 目录~/assets/your_image.png，请在资产前使用斜杠。

    <template>
  <img src="~/assets/your_image.png" />
</template>
    1
    2
    3

    在您的 css 文件中，如果您需要引用您的 assets 目录，请使用~assets/your_image.png（不带斜杠）

    background: url('~assets/banner.svg');
    1

    使用动态图像时，您需要使用 require

    <img :src="require(`~/assets/img/${image}.jpg`)" />
    1

    样式：

    Nuxt.js 允许您定义要全局设置的 CSS 文件/模块/库（包含在每个页面中）。在 nuxt.config 中，您可以使用 CSS 属性轻松添加样式。

    // nuxt.config.js
export default {
  css: [
    // Load a Node.js module directly (here it's a Sass file)
    'bulma',
    // CSS file in the project
    '~/assets/css/main.css',
    // SCSS file in the project
    '~/assets/css/main.scss'
  ]
}

  • static

    该 static 目录直接映射到服务器根目录并包含必须保留其名称（例如robots.txt）或可能不会更改（例如 favicon）的文件

  • nuxt.config.js

    该 nuxt.config.js 文件是 Nuxt.js 的单点配置。如果您想添加模块或覆盖默认设置，这是应用更改的地方。

  • package.json

    该 package.json 文件包含应用程序的所有依赖项和脚本。

# 4. 异步数据 asyncData

  1. 专门用于获取页面服务端渲染的数据

  2. asyncData 只有页面组件中才调用，子组件不会触发调用，，要使用数据的话，需要页面组件传给子组件

  3. asyncData 中没有 this

  4. 当你想要动态页面内容有利于 SEO 或者是提升首屏渲染速度的时候，就在 asyncData 中发请求拿数据

  5. 上下文对象

    • 获取路由参数 context.route.params 或者 context.params
    async asyncData(context){
      const {data} = await axios({
        method:'GET',
        // 静态资源访问：访问的是 static 目录下的 data.json 文件
        url:'http://localhost:3000/data.json'
      })
      const id = JSON.parse(context.params.id)
      // 这里返回的数据会和 data(){} 中的数据合并到一起给页面使用
      return {
        name:data.posts.find(item => item.id === id).name
      }
    }

# 7. nuxt 命令

命令描述
